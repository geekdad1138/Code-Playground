# Code Playground: AI Workflow Exploration

Welcome to the **Code Playground**! This repository serves as an educational knowledge base and personal laboratory dedicated to exploring, documenting, and sharing AI-assisted development methodologies.

## 🎯 Mission
The goal of this repository is to document the "Meta-Work"—how AI tools (like GitHub Copilot, agents, and sub-agents) are steered, orchestrated, and utilized to build software efficiently. It is designed to be both a personal learning space and a valuable resource for the broader developer community.

## 🗂️ What's Inside?

This monorepo is divided into several core pillars of AI engineering:

### [1. Agent Playground](./01-agent-playground/)
The core of AI orchestration. Here you will find documentation and examples on:
- Multi-agent loops and setup.
- Defining sub-agent roles and personas.
- Pattern breakdowns for orchestrating complex tasks.

### [2. Spec-Driven Development](./02-spec-driven-development/)
A core methodology where detailed Markdown specifications act as the blueprint, context anchor, and status tracker for AI agents.
- **Why it matters:** It manages AI context windows and keeps projects on track.
- Includes reusable templates for various task types (feature addition, refactoring, etc.).

### [3. VS Code Power Tools](./03-vscode-power-tools/)
A dedicated section for optimizing the IDE environment for AI workflows.
- "For dummies" summaries of new VS Code features.
- Guides on using VS Code Profiles for strict context isolation between projects.
- Recommended extensions for AI engineers.

### [4. Community Catalog](./04-community-catalog/)
A curated list linking to other open-source agent implementations, tools, and discussions to foster community learning.

## ⚙️ The "Kitchen Equipment"
We document the configuration layers that drive the AI, including:
- **Global Instructions:** Broad rules set in `.github/copilot-instructions.md`.
- **Custom Prompts:** Reusable commands stored in `.github/prompts/`.
- **Agent Configurations:** Files like `agents.md` that define personas and orchestration patterns.
